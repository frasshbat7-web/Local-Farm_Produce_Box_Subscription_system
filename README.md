# نظام اشتراكات صناديق المنتجات الزراعية المحلية
## Local Farm Produce Box Subscription System

نظام ويب مبني بلغة **PHP** وقاعدة بيانات **MySQL** لإدارة اشتراكات العملاء في صناديق المنتجات الزراعية المحلية (خضروات، فواكه، منتجات عضوية...)، بواجهة عربية بالكامل تدعم اتجاه الكتابة من اليمين لليسار (RTL).

---

## 📋 نظرة عامة

يتيح النظام للمشرف (بعد تسجيل الدخول) القيام بما يلي:
- عرض لوحة تحكم تلخص إجمالي المشتركين، المشتركين النشطين، وعدد الاشتراكات الأسبوعية، مع قائمة بأحدث المشتركين المنضمين.
- إدارة كاملة للمشتركين: إضافة، عرض تفاصيل، تعديل، وحذف.
- البحث عن مشترك بالاسم أو البريد الإلكتروني أو العنوان.
- تصنيف الاشتراكات حسب النوع (أسبوعي / شهري / ربع سنوي) والحالة (نشط / موقف / ملغى).

---

## 🚀 خطوات التشغيل (محلياً باستخدام XAMPP)

1. انسخ مجلد المشروع بالكامل إلى `htdocs` (في حال استخدام XAMPP).
2. شغّل **Apache** و **MySQL** من لوحة تحكم XAMPP.
3. افتح **phpMyAdmin** ونفّذ ملف `farm_subscription_db.sql` لإنشاء قاعدة البيانات `farm_subscription_db` والجدول `subscribers` مع بيانات تجريبية.
4. تأكد من إعدادات الاتصال بقاعدة البيانات في `config.php`:
   ```php
   define('DB_SERVER', 'localhost');
   define('DB_USERNAME', 'root');
   define('DB_PASSWORD', '');
   define('DB_NAME', 'farm_subscription_db');
   ```
5. افتح المتصفح على: `http://localhost/Local-Farm_Produce_Box_Subscription_system/login.php`

## 🗄️ هيكل قاعدة البيانات

**قاعدة البيانات:** `farm_subscription_db`
**الجدول:** `subscribers`

| العمود | النوع | الوصف |
|---|---|---|
| id | INT (PK, AUTO_INCREMENT) | المعرّف الفريد للمشترك |
| full_name | VARCHAR(255) | الاسم الكامل للمشترك |
| email | VARCHAR(255) UNIQUE | البريد الإلكتروني (فريد) |
| phone | VARCHAR(20) | رقم الهاتف |
| address | TEXT | العنوان الكامل |
| subscription_type | ENUM | أسبوعي، شهري، ربع سنوي |
| subscription_status | ENUM | نشط، موقف، ملغى |
| start_date | DATE | تاريخ بدء الاشتراك |
| payment_method | ENUM | بطاقة ائتمان، تحويل بنكي، نقدي عند الاستلام |
| box_preferences | TEXT | تفضيلات محتوى الصندوق |
| notes | TEXT | ملاحظات إضافية |
| created_at | TIMESTAMP | تاريخ إنشاء السجل تلقائياً |
| updated_at | TIMESTAMP | تاريخ آخر تعديل تلقائياً |

## ⚠️ ملاحظات أمنية هامة

هذا المشروع **تعليمي/تجريبي** ويحتوي على نقاط يجب معالجتها قبل استخدامه في بيئة إنتاج حقيقية:

1. **حقن SQL جزئي (SQL Injection):** بيانات النماذج (POST) مُعقّمة عبر `mysqli_real_escape_string()` في add/edit، لكن معرّف السجل `id` الممرر عبر الرابط (`$_GET['id']` و `$_GET['delete_id']`) يُستخدم مباشرة في الاستعلامات في `subscribers.php` و `edit_subscriber.php` و `view_subscriber.php` دون تعقيم أو تحويل صريح لنوع رقمي.
2. **عدم تعقيم المخرجات (XSS):** بيانات مثل `full_name` و `address` تُطبع مباشرة في HTML دون `htmlspecialchars()`.
3. **بيانات دخول ثابتة:** اسم المستخدم وكلمة المرور مكتوبة مباشرة في الكود (Hard-coded) وبدون تشفير (Hashing).
4. **الحذف عبر رابط GET:** حذف المشترك يتم عبر رابط مباشر (`subscribers.php?delete_id=`) دون حماية CSRF.

**التوصية:** التحقق من أن `id` رقم صحيح عبر `(int)` أو Prepared Statements في جميع الاستعلامات، وتطبيق `htmlspecialchars()` على المخرجات، واستخدام `password_hash()` لإدارة كلمات المرور.
